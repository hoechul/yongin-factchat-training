3 Principles

2. 정확도를 높이는 프롬프트 3원칙

① 역할 부여
"당신은 지방자치법에 능통한 행정 전문가입니다"
② 맥락 제공
부서 · 목적 · 독자 · 분량 · 톤을 명시
③ 근거 요구
"관련 법령 조항과 출처 URL을 함께 표기"
→ "공고문 써줘"보다 정확도·신뢰도가 크게 향상되어 재작성이 거의 필요 없습니다.
Framework

3. C.R.E.A.T.E. 프레임워크 — 좋은 질문을 만드는 6단계

C
Context
배경·상황
"○○조례 개정 관련"
R
Role
역할 지정
"기획팀 주무관"
E
Example
예시 제시
기존 문서 붙여넣기
A
Action
지시 동작
"3문단으로 작성"
T
Tone
톤·문체
"공문 격식체"
E
Evaluation
검증 요구
"근거·출처 표기"
당신은 용인시청 기획팀 주무관입니다.(R) 첨부한 회의 내용을 바탕으로(C·E) 보도자료 초안을 3문단·800자 내외로 작성하되(A), 공공기관 보도자료 격식체로 쓰고(T), 인용한 통계는 출처 URL을 함께 표기하세요.(E)
→ 6요소를 한 번에 지정해 재작성 없이 바로 배포 가능한 초안이 나옵니다.

4. 크레딧 절약 팁

  • 모델 선택 전략 — 단순 작업은 경량, 심층 분석만 상위 모델 사용
  • 긴 대화 분리 — 프로젝트 기능으로 주제별 분리, 컨텍스트 재사용
  • 프로젝트 지침 저장 — 자주 쓰는 역할 문장을 저장해 매번 다시 입력하지 않기
  • 재활용 — 실패한 답변은 "수정 재생성", 주제가 바뀌면 "새 대화"
🧪 20쪽 보고서라면 오탈자·문단 정리는 경량 모델로 먼저, 정책 논리 검토만 상위 모델로 → 같은 결과에 크레딧 소모 절반 이하.
Checklist

✅ 답변 신뢰도 체크리스트

  • 법령·조례 답변은 조문 번호와 출처 URL 확인
  • 웹 검색을 켜고 근거 링크를 직접 열어 교차 확인
  • 수치·날짜·인명은 원문과 한 번 더 대조
  • 최종본은 담당자가 사실관계·표현 최종 검수
