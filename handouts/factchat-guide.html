정책 분석 · 보고서 · 조례 검토상위 모델
이미지 · PPT · 복합 자동화Super Agent
  • 긴 대화 분리 — 프로젝트 기능으로 주제별 분리, 컨텍스트 재사용
  • 재활용 — 실패한 답변은 "수정 재생성", 주제가 바뀌면 "새 대화"

🎯 바로 써먹는 실습 시나리오

  • 보도자료 초안 3분 만에 작성
  • 조례안 요약 + 타 지자체 비교표 만들기
  • 민원 답변서 톤 조정 (딱딱 → 공감형)
  • 회의록 → 실행과제 목록 자동 정리
▶ 각 시나리오의 상황·복사용 프롬프트·결과 예시는 교육 페이지 「행정 업무 시나리오 실습」에서

✅ 답변 신뢰도 체크리스트

  • 법령·조례 답변은 조문 번호와 출처 URL 확인
  • 웹 검색을 켜고 근거 링크를 직접 열어 교차 확인
  • 수치·날짜·인명은 원문과 한 번 더 대조
  • 최종본은 담당자가 사실관계·표현 최종 검수

1 20쪽 정책보고서 다듬기 — 경량·상위 모델 나눠 쓰기 약 10분
상황
부서에서 작성한 20쪽 분량 정책보고서 초안을 상급자 보고 전에 다듬어야 합니다. 처음부터 끝까지 상위 모델로(×) 돌리면 크레딧이 금방 소진됩니다. → 단순 정리는 경량 모델, 판단이 필요한 부분만 상위 모델로 나눠 작업합니다.
1 경량
맞춤법·오탈자
전체 본문 교정
2 경량
문단 구조 정리
4단 구조로 재배치
3 상위
정책 논리 검토
법령 충돌·비약 확인
4 경량
보고용 요약
5줄 핵심 정리
STEP 1 · 경량 모델 맞춤법·오탈자 교정
다음 보고서 초안의 맞춤법·오탈자·띄어쓰기만 교정해줘. 내용과 문장 구조는 바꾸지 말고, 고친 부분은 표시해줘.
[보고서 본문 전체 붙여넣기]
STEP 2 · 경량 모델 문단 구조 정리
위 교정본을 「현황 → 문제점 → 개선방안 → 기대효과」 4단 구조로 문단을 재배치하고, 각 문단에 소제목을 붙여줘. 문장 표현은 그대로 유지해줘.
STEP 3 · 상위 모델 정책 논리 검토 (여기서만 상위 모델 사용)
당신은 지방행정 정책 전문가입니다. 아래 「개선방안」 부분이 상위 법령·조례와 충돌하지 않는지, 논리적 비약이나 근거 부족은 없는지 검토하고, 문제가 있으면 관련 조항과 함께 수정 의견을 제시해줘.
[개선방안 문단만 붙여넣기]
STEP 4 · 경량 모델 보고용 요약
위 최종 보고서를 상급자 보고용 5줄 요약으로 정리해줘. 핵심 수치는 그대로 살려줘.
✔ 결과
  • 상위 모델은 STEP 3(전체의 약 1/5 분량)에만 사용 → 크레딧 소모 절반 이하
  • 같은 품질의 결과물을 얻으면서 월말 크레딧 소진 없이 사용 가능
💡 Tip — STEP마다 모델 선택을 바꾸는 걸 잊지 마세요. 대화 입력창의 모델 선택에서 경량(기본) ↔ 상위를 전환합니다. 주제가 이어지면 같은 대화에서, 완전히 다른 작업이면 새 대화로 시작하세요.
2 주민 공고문 작성 — 프롬프트 3원칙 적용하기 약 3분
상황
주민에게 배포할 공고문을 써야 합니다. "공고문 써줘"처럼 막연히 요청하면 부정확하고 딱딱한 초안이 나와 여러 번 다시 써야 합니다. → ①역할 · ②맥락 · ③근거 세 가지를 한 문장에 담아 요청합니다.
① 역할
"지방자치법에 능통한 행정 전문가"
② 맥락
독자·목적·분량·톤 명시
③ 근거
조례 조항·출처 URL 요구
✖ 이렇게 쓰면 (막연한 요청)
폐기물 무단투기 관련 공고문 써줘.
✔ 이렇게 쓰세요 · 복사해서 사용 (3원칙 적용)
당신은 지방자치법에 능통한 행정 전문가입니다.(①역할) [생활폐기물 무단투기 금지]를 알리는 주민 대상 공고문을, 노년층도 이해하기 쉬운 말로 500자 내외, 공공기관 공고문 격식체로 작성해줘.(②맥락) 인용한 조례 조항 번호와 출처 URL(law.go.kr)을 함께 표기해줘.(③근거)
✔ 결과
  • 역할·맥락·근거를 지정해 바로 배포 가능한 수준의 공고문이 한 번에 나옴
  • 조항·출처가 붙어 사실 확인이 쉽고 신뢰도가 높아 재작성이 거의 필요 없음
💡 Tip — [대괄호] 부분만 실제 안건으로 바꾸면 다른 공고문에도 그대로 재사용할 수 있습니다. 근거가 붙은 답변은 링크를 직접 열어 조문 원문과 한 번 더 대조하세요.
3 보도자료 초안 작성 — C.R.E.A.T.E. 6요소 한 번에 지정하기 약 5분
상황
행사·정책 관련 보도자료 초안을 만들어야 합니다. 요청에 필요한 정보를 빠뜨리면 톤·분량·근거가 제각각인 초안이 나와 여러 번 고쳐 써야 합니다. → C·R·E·A·T·E 6요소를 한 프롬프트에 모두 담아 재작성 없이 바로 배포 가능한 초안을 받습니다.
C
Context
회의 내용 첨부
R
Role
기획팀 주무관
E
Example
회의록을 근거로
A
Action
3문단·800자
T
Tone
보도자료 격식체
E
Evaluation
통계 출처 URL
STEP 1 · 회의 자료 첨부 (파일 업로드 또는 본문 붙여넣기)
보도자료의 근거가 될 회의록·행사 계획서·통계 자료를 대화에 먼저 첨부하거나 붙여넣습니다. (Context·Example)
STEP 2 · 6요소 프롬프트 · 복사해서 사용
당신은 용인시청 기획팀 주무관입니다.(R·역할) 첨부한 회의 내용을 바탕으로(C·E·맥락/예시) 보도자료 초안을 제목·리드문·본문 3문단·문의처 형식, 800자 내외로 작성하되(A·지시), 공공기관 보도자료 격식체로 써줘.(T·톤) 인용한 통계·수치는 출처 URL을 함께 표기해줘.(E·검증)
✔ 결과
  • 6요소를 한 번에 지정해 제목·리드·본문·문의처가 갖춰진 초안이 한 번에 완성
  • 톤·분량·근거가 정확히 맞아 재작성 없이 바로 배포 가능한 수준
💡 Tip — 6요소를 모두 넣기 어렵다면 최소한 R(역할)·A(지시)·E(검증) 3가지만이라도 지정하세요. 결과가 아쉬우면 "수정 재생성"으로 톤·분량만 조정하면 크레딧을 아낄 수 있습니다.
