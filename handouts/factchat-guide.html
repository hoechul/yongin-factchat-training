1 크레딧을 아끼는 모델 선택 전략

업무 유형추천 모델
요약 · 번역 · 정리 · 맞춤법경량(기본) 모델
정책 분석 · 보고서 · 조례 검토상위 모델
이미지 · PPT · 복합 자동화Super Agent
  • 긴 대화 분리 — 프로젝트 기능으로 주제별 분리, 컨텍스트 재사용
  • 재활용 — 실패한 답변은 "수정 재생성", 주제가 바뀌면 "새 대화"
  • 단순 작업에 경량 모델만 써도 크레딧 3~5배 절약
📌 예시 20쪽 보고서 작업 시 전체를 상위 모델로(×) 돌리지 말고, 오탈자·문단 정리는 경량 모델로 먼저 하고 정책 논리 검토만 상위 모델로. → 같은 결과에 크레딧 소모 절반 이하, 월말 소진 없이 사용

2 정확도를 높이는 프롬프트 3원칙

① 역할 부여
"당신은 지방자치법에 능통한 행정 전문가입니다"
② 맥락 제공
부서·목적·독자·분량·톤을 명시
③ 근거 요구
"관련 법령 조항과 출처 URL을 함께 표기"
📌 예시(3원칙 적용) "당신은 지방자치법에 능통한 행정 전문가입니다(①역할). 주민 대상 공고문을 쉬운 말 500자로 작성하고(②맥락), 인용한 조례 조항과 출처 URL을 표기하세요(③근거)." → "공고문 써줘"보다 정확도·신뢰도가 크게 향상, 재작성 불필요

3 C.R.E.A.T.E. 프레임워크 — 좋은 질문을 만드는 6단계

C
Context
배경·상황
"○○조례 개정 관련"
R
Role
역할 지정
"기획팀 주무관"
E
Example
예시 제시
기존 문서 붙여넣기
A
Action
지시 동작
"3문단으로 작성"
T
Tone
톤·문체
"공문 격식체"
E
Evaluation
검증 요구
"근거·출처 표기"
📌 예시 프롬프트 ▸ "당신은 용인시청 기획팀 주무관입니다.(Role) 첨부한 회의 내용을 바탕으로(Context·Example) 보도자료 초안을 3문단·800자 내외로 작성하되(Action), 공공기관 보도자료 격식체로 쓰고(Tone), 인용한 통계는 출처 URL을 함께 표기하세요.(Evaluation)" → 6요소를 한 번에 지정해 재작성 없이 바로 배포 가능한 초안이 나옵니다.

🎯 바로 써먹는 실습 시나리오

  • 보도자료 초안 3분 만에 작성
  • 조례안 요약 + 타 지자체 비교표 만들기
  • 민원 답변서 톤 조정 (딱딱 → 공감형)
  • 회의록 → 실행과제 목록 자동 정리
▶ 각 시나리오의 상황·복사용 프롬프트·결과 예시는 교육 페이지 「행정 업무 시나리오 실습」에서

✅ 답변 신뢰도 체크리스트

  • 법령·조례 답변은 조문 번호와 출처 URL 확인
  • 웹 검색을 켜고 근거 링크를 직접 열어 교차 확인
  • 수치·날짜·인명은 원문과 한 번 더 대조
  • 최종본은 담당자가 사실관계·표현 최종 검수
